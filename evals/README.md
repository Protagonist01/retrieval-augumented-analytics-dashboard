This folder contains the evaluation harness for the Retrieval-Augmented Analytics (RAA) Dashboard. The evaluation measures how accurately the system translates natural language questions into valid DuckDB SQL queries that execute and produce the correct output.

---

## Metric Definitions

The script `evals/run_evals.py` measures the following metrics:

1. **Execution Accuracy** â€” The percentage of questions where the generated SQL yields the exact same rows (order-insensitive) as the ground-truth SQL query. This is the primary success metric.
2. **SQL Validity Rate** â€” The percentage of generated queries that pass syntax, structural, and safety checks without throwing errors.
3. **P50 & P95 Latency** â€” The median and 95th percentile times required to process a natural language query end-to-end (SQL generation, validation, execution, and explanation).

---

## Dataset Schema

The evaluation set targets the 4-table e-commerce database:
- **`customers`** (`customer_id`, `name`, `email`, `city`, `country`, `signup_date`)
- **`products`** (`product_id`, `name`, `category`, `subcategory`, `cost_price`, `list_price`)
- **`orders`** (`order_id`, `customer_id`, `order_date`, `status`, `payment_method`, `shipping_city`, `shipping_country`, `total_amount`)
- **`order_items`** (`item_id`, `order_id`, `product_id`, `quantity`, `unit_price`, `discount_pct`)

The dataset is deterministically generated by `data/generate_sample.py`.

## How to Run Evaluations

You can run evaluations using the top-level `Makefile`:

```bash
# Run evaluations against the default model configured in .env
make eval
```

Alternatively, override the model settings in your environment variables:
```bash
# Run against a different OpenAI model
OPENAI_MODEL=gpt-4-turbo python evals/run_evals.py

# Run against Ollama locally
LLM_PROVIDER=ollama OLLAMA_MODEL=sqlcoder:7b python evals/run_evals.py
```

1. Open [golden_set.jsonl](file:///d:/document/dev/work/portfolio%20projects/raa-dashboard/evals/golden_set.jsonl).
2. Add a new JSON line in the following format:
   ```json
   {"id": "ec_081", "nl": "Show total revenue of all electronics", "expected_sql": "SELECT SUM(oi.quantity * oi.unit_price) FROM order_items oi JOIN products p ON oi.product_id = p.product_id WHERE p.category = 'Electronics'", "expected_row_count_min": 1, "difficulty": "easy"}
   ```
3. Run the evaluation suite to confirm execution accuracy.

## Dialect Differences (DuckDB vs Spider/SQLite)

This dashboard uses **DuckDB** which introduces minor syntax enhancements over standard Spider benchmark (SQLite) queries:
- **`STRFTIME`** is used for formatting dates. E.g., `STRFTIME(order_date, '%Y-%m')`.
- **`YEAR` / `MONTH`** functions exist as native functions. E.g., `YEAR(order_date) = 2024`.
- **`INTERVAL`** keyword for date addition/subtraction: `order_date + INTERVAL 5 DAY`.
